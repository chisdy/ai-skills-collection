# 工作流模式参考

## 常见项目类型的工作流模式

### 1. 新功能开发

**研究重点**: 现有架构分析、依赖关系梳理
**构思策略**: 渐进式 vs 重构式实现
**计划要素**: 数据库迁移、API 设计、前端组件
**执行顺序**: 后端 → API → 前端 → 测试

### 2. 性能优化

**研究重点**: 性能瓶颈识别、监控数据分析
**构思策略**: 缓存策略 vs 架构重构 vs 算法优化
**计划要素**: 基准测试、优化实施、效果验证
**执行顺序**: 基准 → 优化 → 测试 → 监控

### 3. 技术债务重构

**研究重点**: 代码质量评估、依赖关系分析
**构思策略**: 渐进式重构 vs 大规模重写
**计划要素**: 风险评估、回滚策略、测试覆盖
**执行顺序**: 测试 → 重构 → 验证 → 部署

### 4. 第三方集成

**研究重点**: API 文档研究、兼容性分析
**构思策略**: 直接集成 vs 适配器模式 vs 中间件
**计划要素**: 认证配置、错误处理、数据映射
**执行顺序**: 配置 → 适配 → 测试 → 监控

## 决策树模式

### 技术选型决策

```
复杂度评估 → 团队技能 → 维护成本 → 性能要求 → 最终选择
```

### 架构设计决策

```
业务需求 → 非功能需求 → 约束条件 → 架构模式 → 技术栈
```

### 优化策略决策

```
问题识别 → 根因分析 → 解决方案 → 成本效益 → 实施计划
```
