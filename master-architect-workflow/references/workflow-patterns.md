# 工作流模式参考

## 常见项目类型的研究重点

| 项目类型 | 研究重点 | 构思策略 | 执行顺序 |
|---------|---------|---------|---------|
| 新功能开发 | 现有架构、依赖关系 | 渐进式 vs 重构式 | 后端 → API → 前端 → 测试 |
| 性能优化 | 瓶颈识别、监控数据 | 缓存 vs 架构重构 vs 算法优化 | 基准 → 优化 → 测试 → 监控 |
| 技术债务重构 | 代码质量、依赖分析 | 渐进式重构 vs 大规模重写 | 测试 → 重构 → 验证 → 部署 |
| 第三方集成 | API 文档、兼容性 | 直接集成 vs 适配器模式 | 配置 → 适配 → 测试 → 监控 |

---

## 决策树

### 技术选型

```
复杂度评估 → 团队技能 → 维护成本 → 性能要求 → 最终选择
```

### 架构设计

```
业务需求 → 非功能需求 → 约束条件 → 架构模式 → 技术栈
```

### 优化策略

```
问题识别 → 根因分析 → 解决方案 → 成本效益 → 实施计划
```
